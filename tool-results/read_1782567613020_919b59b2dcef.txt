    56‚Üí    56‚Üí2. NON-NEGOTIABLE IMPLEMENTATION RULES
    57‚Üí    57‚Üí======================================================================
    58‚Üí    58‚Üí
    59‚Üí    59‚ÜíFollow these rules throughout the development process:
    60‚Üí    60‚Üí
    61‚Üí    61‚Üí1. Do not build the complete application in a single file.
    62‚Üí    62‚Üí2. Do not mix frontend, backend, database, and configuration logic.
    63‚Üí    63‚Üí3. Do not hardcode admin passwords, JWT secrets, database credentials,
    64‚Üí    64‚Üí   storage keys, email credentials, or production URLs.
    65‚Üí    65‚Üí4. Do not expose sensitive data in frontend bundles or API responses.
    66‚Üí    66‚Üí5. Do not rely only on frontend validation.
    67‚Üí    67‚Üí6. Every sensitive rule must also be enforced by the backend.
    68‚Üí    68‚Üí7. Never trust user-provided role, user ID, score, test duration,
    69‚Üí    69‚Üí   batch assignment, attempt count, or video completion values.
    70‚Üí    70‚Üí8. Do not expose correct MCQ answers before an attempt is submitted.
    71‚Üí    71‚Üí9. Do not calculate final test scores on the client.
    72‚Üí    72‚Üí10. Do not allow blocked, rejected, pending, or inactive students to
    73‚Üí    73‚Üí    access protected learning content.
    74‚Üí    74‚Üí11. Do not allow a student to access another student's profile,
    75‚Üí    75‚Üí    results, batches, materials, attempts, or activity.
    76‚Üí    76‚Üí12. Do not allow deleted or unpublished content to remain accessible
    77‚Üí    77‚Üí    through direct API requests.
    78‚Üí    78‚Üí13. All destructive actions must require confirmation.
    79‚Üí    79‚Üí14. All asynchronous actions must display loading, success, and error states.
    80‚Üí    80‚Üí15. Prevent duplicate form submission and duplicate database records.
    81‚Üí    81‚Üí16. Use reusable components, services, validation schemas, and error handlers.
    82‚Üí    82‚Üí17. Use database transactions for critical multi-step operations.
    83‚Üí    83‚Üí18. Create database migrations and seed scripts.
    84‚Üí    84‚Üí19. Include automated tests for critical workflows.
    85‚Üí    85‚Üí20. Do not claim that an embedded YouTube URL is impossible to discover.
    86‚Üí    86‚Üí    Browser-based video URLs may still be inspected by technically skilled
    87‚Üí    87‚Üí    users. Reduce exposure through server-controlled video IDs,
    88‚Üí    88‚Üí    privacy-enhanced embeds, referrer policies, access validation, and UI
    89‚Üí    89‚Üí    restrictions, but do not make technically false security claims.
